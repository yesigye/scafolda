Installation

Step One

Unzip the contents of the archive file into your server web directory usually the www, http or the public folder.

Step Two

Edit the the base url in the config file to point to your application's url. Remember to keep the trailing slash. 

$config['base_url'] = "http://scafolda.test/";
You will find the config file at application/config/config.php

Step Three

Edit the .htaccess file at root of your app installation. You must have mod_rewrite enabled on your server for this step to work.
Depending on the location of the application directory on your server, in the .htaccess file, edit this line as follows;

  1. If your app is at the root of your server directory. This is the default setting 
    RewriteBase /
  2. If your app is in a directory (assuming your directory name is "admin") 
    RewriteBase /admin/
  3. For subsequent directories, set the path to your app directory. Remember the trailing slash. 
    RewriteBase /directory/sub-directory/

Step Three

Connecting to your database.

Scafolda will prompt you to enter your database connection values at the login page. This is the default method of authentication. If you do not want to change this, then this step is not required.

However if you want to disable this authentication option, you must set database values in the database config file at application/config/database.php
Database connection example
'hostname' => 'localhost',
        'username' => 'demo_user',
        'password' => 'my_secret',
        'database' => 'demo_database',
Additional configuration can be found in the database config file.

To read more about Scafolda authentication, go to authentication section

Authentication

The script uses your database connection as the default method of authentication. At the login prompt, you will enter your database server's hostname, the database you are connecting to and your server's username and password.

Displaying data

Scafolda offers a covenient way to access and quickly manipulate data in your tables. Data is presented in a tabular design which makes it easy to read and understand. Scafolda employees unique features like character limiters, thumbnail generators and field detection to ensure that the data is presented in a simple and yet relevant way.

For example, Scafolda will know to present image url strings as thumbnails, trucante long sentences or descriptions that might not fit on the viewport, recognize boolean data types and intergers, map table relationships and more. This makes for an intuitive table that is compact and easy to read.

Generating Data

Scafold uses special providers to add fake data into your table. In the previous section, we saw how data is added through a more traditional form filling style where you explicitly enter column data. While generating data however, fake data will be randomly entered depending on the data type of the column. You can generate data in one of two ways;

  1. Auto Selection

    The system decides on what kind of data to generate for each field in the table. It achieves this by analyzing the column type, constraints and even name to determine what type of data is expected. This means that even though data is generated randomly, it remains true to your schema.

    For auto selection, you only need to define the number of rows that you want to generate.

  2. Manual Selection

    Scafolda give you the option to specify the type of data that you want to generate for every column. There is a huge pool of over 50 data types that you can choose from to represent your data perfectly. However, the actual data generated that is still randomly generated. If you want to enter data directly, use the traditional forms instead of data type generators.

Deleting Data

Rows can be deleted by clicking the delete icon × coveniently placed at the end of every row.

Batch delete
Deleting in bulk is also supported with the batch deleting feature. To delete multiple rows; select the rows that you want to delete by clicking on the rows or clicking the checkboxes at the front of every row. Once you have checked your rows, click the delete button in the toolbar at the top of the table.
How delete works

Deleting rows from the database is achieved in one of two ways; deleting by a unique column or deleting by row position. When deleting rows, the system checks for a unique key in the table and uses that column as a reference to delete the row. If the table does not have any unique keys, the row will be deleted by its counting position in the table, this is especially useful since not all tables have an "id".

When deleting a row from a table by position, the where clause uses all columns for reference, this means that duplicate rows are also deleted.
